# Висновки щодо швидкості алгоритмів пошуку підрядка
 
## Опис завдання
 
Виконано порівняльний аналіз трьох алгоритмів пошуку підрядка у тексті:
- **KMP** (Кнута–Морріса–Пратта)
- **Boyer-Moore** (Боєра–Мура)
- **Rabin-Karp** (Рабіна–Карпа)

Пошук проводився у двох текстових файлах:
- `стаття 1.txt` — стаття про використання алгоритмів у бібліотеках мов програмування
- `стаття 2.txt` — стаття про методи та структури даних для рекомендаційних систем

Для кожного файлу тестувалося два підрядки:
- **"видати решту суми"**
- **"являється побудов"**

## Аналіз результатів
 
### Найшвидший алгоритм для `стаття 1.txt`
 
**Boyer-Moore** — найшвидший алгоритм для обох типів пошуку:

## Загальні висновки
 
### 1. Boyer-Moore — найшвидший алгоритм в обох текстах
 
Алгоритм Боєра–Мура показав найкращі результати в усіх чотирьох сценаріях тестування. Це пояснюється його ключовою особливістю: алгоритм переглядає текст **справа наліво** і може пропускати значні ділянки тексту. При роботі з природними мовами (кирилиця, великий алфавіт) ці пропуски особливо ефективні, що й забезпечило беззаперечну перевагу в обох тестових файлах.
 
### 2. KMP — стабільний середній результат
 
Алгоритм Кнута–Морріса–Пратта показав середні результати. Він гарантує лінійний час O(n + m), не повертається назад у тексті та є передбачуваним.
 
### 3. Rabin-Karp — найповільніший алгоритм
 
Алгоритм Рабіна–Карпа виявився найповільнішим у всіх тестах — в середньому у 8–13 разів повільніше за Boyer-Moore. Це пов'язано з накладними витратами на обчислення **хеш-функції** для кожної позиції у тексті.
 
### 4. Особливість пошуку неіснуючих підрядків
 
Цікаво, що Boyer-Moore показує особливо великі переваги саме при пошуку **відсутніх підрядків**: алгоритм швидко відхиляє невідповідності й проходить весь текст з великими стрибками, тоді як KMP і Rabin-Karp змушені перебирати позиції більш послідовно.
